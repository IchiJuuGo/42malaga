# Iniciamos git EN EL DIRECTORIO
git init

# AÃ±adimos todos los archivos del repositorio local
git add .

# Capturamos una instantanea de los cambios que han
habido en el repositorio local y le ponemos nombre
git commit -m "<fecha de hoy>"

# Subimos los cambios a GitHub

### EN LA CONSOLA ###
>> En la carpeta del proyecto que vaya a entregar:
git init (si no esta hecho en esa carpeta, se comprueba con (cmd + shift + .) para ver los archivos ocultos)
git remote add origin <enlace repositorio git de la intra> (origin es el nombre que le damos)
git add .
git status para comprobar
git rm -f .DS_Store si aparece
git status para comprobar
git commit -m "Done"
git push origin master

>> Para comprobar que esta subido bien hacemos el clone a una carpeta de prueba
git clone <enlace repositorio git de la intra> .


Tambien tenemos tambien para comprobar los procesos:
git status
git log
git remote
